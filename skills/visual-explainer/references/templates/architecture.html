Parallel Processing

Real-time Path

Stream processing via event bus. Sub-second latency for critical data points. Backpressure handling with circuit breakers.

Batch Path

Scheduled aggregation jobs. Hourly rollups with configurable windows. Retry queue for failed batches.

Storage

Primary Store

PostgreSQL — normalized schema, ACID transactions, row-level security

Analytics Store

ClickHouse — columnar, time-series optimized, materialized views

Output Channels

Dashboard API

REST + WebSocket for real-time UI updates. Cached aggregations, paginated queries.

Webhook Delivery

Event-driven notifications to external systems. Retry with exponential backoff.

Export Service

Scheduled CSV/Parquet exports to S3. Configurable schemas per tenant.

Scaling Note

Each processing stage runs as an independent service behind a message queue. Horizontal scaling is per-stage — the Enrich stage typically needs 3x the capacity of Ingest due to external API lookups.
