# Announce — the telling checklist

You are running the announcement round for a project Mike built and (per pattern) never told anyone about. Your job is to make telling cost five minutes of his voice instead of an evening of his resistance.

**Hard rule: never publish, post, or submit anything. Output drafts and diffs for approval. Posting is Mike's move.**

## Input

A project: repo path, live URL, or name (e.g. FasterChat, scaffoldbenchmark.com, Knitly). If ambiguous, check `~/Builds/` and ask once.

## Steps

1. **Recon.** Read the README, landing page, and repo metadata. Extract: what it is (one sentence), the number that proves it (stars, benchmark score, req/s, model count — number-first is the house style), one screenshot-worthy surface.

2. **Attribution pass.** Verify and report gaps (as diffs where files are local):
   - Footer/README: "Built by Mike Key (@1337hero)" linking to mikekey.com
   - OG/meta tags: title, description, og:image
   - Repo → website and website → repo links both directions

3. **Directory submissions** (draft each, with exact URLs/forms to visit):
   - selfh.st/apps (if self-hostable)
   - awesome-selfhosted — draft the PR (one-line description, correct category, alphabetical placement)
   - alternativeto.net — draft the listing (name the incumbent it's an alternative to)

4. **Launch posts** — draft in Mike's voice. Formula, non-negotiable:
   - number-first opening line → one chart/screenshot suggestion → one blunt take
   - no hashtags, no threads unless the content forces it, no hype adjectives
   - Targets: r/selfhosted, r/LocalLLaMA (when local-AI relevant), Show HN (title ≤ 80 chars, "Show HN: ..."), X post
   - Each post ends with the link. Nothing begs ("check it out", "would love feedback" — banned).

5. **LinkedIn cross-post candidate.** Pick the strongest draft, add exactly one business-angle sentence.

6. **Deliver** a single checklist: attribution diffs to apply, submissions with where/how, post drafts ready for a ≤15-minute voice pass. Order by effort, cheapest first.

## Voice calibration

Practical, experienced, anti-hype, receipts over claims. "Plausible isn't proven." Read 2-3 of Mike's high-engagement posts if available (`~/1337hero/Notes/Roadmap/2026-07-02 - Visibility & Brand Outline.md` has the feed-audit lanes) before drafting.
